# Internacionalización (complementa AGENTS.md)

Las reglas base de i18n están en `AGENTS.md` sección "Internationalization".
Este archivo agrega detalles operativos.

## Namespaces actuales
- `common` — vocabulario compartido (botones, labels, general)
- `auth` — login, sign-up, sign-out
- `nav` — labels de navegación
- `settings` — texto de UI de settings
- Archivos: `frontend/src/locales/<lang>/<namespace>.json`

## Preferencia de Idioma
- localStorage key: `fincore_lang` (con fallback a navigator).
- **Gap conocido:** la detección actual en `i18n.ts` usa solo `["navigator"]` sin lookup de localStorage. Ver `AI_CONTEXT.md` sección 8.

## Agregar un Nuevo Namespace
1. Crear `frontend/src/locales/es/<namespace>.json` y `.../en/<namespace>.json`
2. Importar ambos en `frontend/src/lib/i18n.ts`
3. Agregar al objeto `resources` y al array `ns` en `init()`
4. Usar: `const { t } = useTranslation("<namespace>")`
